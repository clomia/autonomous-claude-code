작업한 내용을 철저히 검수해.

검수 항목:

- 논리적으로 무결한가?
- 불필요한 복잡성이 있는가?
- 구조적으로 최적화 여지가 있는가?
- 고려하지 못한 경우의 수가 있는가?

검수는 매우 중요하므로 작업의 완성도를 위해서 이전에 검수했음에도 다시 수행할 수 있습니다.
overengineering되지 않도록 주의해야 하므로 문제 발견 시 그것을 수정하는게 적절한지 재검토하세요.

> **여기서 overengineering이란 코드 수정 정도를 의미하는게 아니라 최종 코드 아키텍처 전체를 의미합니다.** 기존 코드를 포함한 최종 결과물의 아키텍처가 중요합니다.

- 철저히 검수한 결과, 이미 최적이라고 판단되면 수정하지 마세요.
- 만약 수정이 필요한 경우 이슈를 종합해서 어떻게 수정하는게 가장 최적일지 충분히 고찰하세요. 그리고 고찰을 통해 나온 수정안이 실제로 유효한 솔루션인지 생각하세요. 최적의 솔루션이 도출되면 코드에 적용하세요.

최대한의 sequential-thinking을 권장합니다.
